# Task 5 — Web UI (R5)

## Опис
Streamlit multipage дашборд поверх R2–R4. Максимум готових компонентів (st.plotly_chart, st.dataframe, st.sidebar), мінімум власного коду.

## Обсяг
- `app/Home.py`: опис проблеми/проєкту, глобальні фільтри (регіон, період) у `st.sidebar`, кеш даних `@st.cache_data`, ключові метрики (KPI).
- `app/pages/1_Огляд.py` (Рівень 1): KPI, топ-регіони, heatmap година×день, тренд кількості.
- `app/pages/2_Тренд_і_сезонність.py` (Рівень 2): STL-графіки, change-points.
- `app/pages/3_Аномалії.py` (Рівень 3): ряд із позначеними сплесками, таблиця аномалій.
- `app/pages/4_Прогноз.py` (Рівень 4): Prophet прогноз з довірчим інтервалом, MAE.
- `app/pages/5_Патерни.py` (Рівень 5): кластери регіонів, пікові години.
- `app/pages/6_Мультиагентний_звіт.py` (Рівень 6): кнопка «Запустити аналіз» → `orchestrator.analyze(...)`, показ findings агентів + оцінок судді (confidence), статус джерела (claude/fallback).

## Критерії готовності
- `streamlit run app/Home.py` стартує без помилок.
- Кожна сторінка рендериться з графіками без трейсбеків.
- Фільтри регіон/період впливають на всі сторінки (спільний стан через session_state/кеш).
