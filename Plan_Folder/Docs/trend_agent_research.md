# Дослідження: Агент тренду та сезонності (Рівень 2)

> Декомпозиція денного ряду кількості тривог на тренд + сезонність + залишок.
> Стек: `statsmodels` (BSD-3, не AGPL), опційно `ruptures` (BSD-2, не AGPL).

## Метод

STL-декомпозиція (Seasonal-Trend decomposition using LOESS) денного часового ряду
кількості тривог по регіону. STL стійка до викидів, гнучка до зміни сезонної амплітуди
і добре працює на денних даних із тижневою сезонністю.

Розкладаємо ряд `y(t)` на три адитивні компоненти:

```
y(t) = trend(t) + seasonal(t) + resid(t)
```

- `trend` — згладжена довгострокова динаміка (зростання/спад інтенсивності тривог);
- `seasonal` — повторюваний тижневий патерн (period=7);
- `resid` — залишок (далі його використовує Агент аномалій, Рівень 3).

Опційно — пошук точок зміни режиму (change points) у компоненті тренду методом
PELT (Pruned Exact Linear Time) з `ruptures`, щоб виявити дати різкої зміни рівня тривог
(напр. початок нової фази бойових дій).

## Бібліотека/виклик

```python
import pandas as pd
import numpy as np
from statsmodels.tsa.seasonal import STL

# series: pd.Series із DatetimeIndex (денна частота), значення = кількість тривог за добу
# Гарантуємо суцільну денну частоту (пропуски -> 0 тривог за добу)
series = series.asfreq("D").fillna(0.0)

stl = STL(series, period=7, robust=True)
res = stl.fit()

trend = res.trend          # pd.Series
seasonal = res.seasonal    # pd.Series
resid = res.resid          # pd.Series

# Напрямок тренду: нахил лінійної регресії по компоненті тренду
x = np.arange(len(trend))
slope = np.polyfit(x, trend.values, 1)[0]   # одиниць тривог на день
direction = "зростання" if slope > 0 else ("спад" if slope < 0 else "стабільно")
```

Опційно — точки зміни режиму (PELT):

```python
import ruptures as rpt

signal = trend.values.reshape(-1, 1)
algo = rpt.Pelt(model="rbf").fit(signal)
# pen — штраф; більший pen -> менше точок зміни
bkps = algo.predict(pen=10)        # індекси кінців сегментів (останній = len(signal))
change_dates = trend.index[[b - 1 for b in bkps[:-1]]]  # дати точок зміни
```

## Входи/виходи

**Входи:**
- `series` — `pd.Series` з `DatetimeIndex` денної частоти; значення — кількість тривог за добу для одного регіону (агрегація з датасету Vadimkin, UTC -> доба).
- `period=7` — тижнева сезонність.

**Виходи:**
- `trend`, `seasonal`, `resid` — три `pd.Series` тієї ж довжини й індексу, що й вхід.
- `slope` (float) і `direction` (str) — кількісний і словесний підсумок напрямку тренду.
- Опційно `change_dates` — список дат точок зміни режиму.

Для UI (Plotly) повертати DataFrame з колонками `[observed, trend, seasonal, resid]`.

## Параметри

| Параметр | Значення | Пояснення |
|---|---|---|
| `period` | `7` | Тижнева сезонність денного ряду. |
| `robust` | `True` | Стійкість до викидів/сплесків (downweighting залишків через біквадратні ваги). |
| `seasonal` | `7` (за замовч., непарне ≥7) | Вікно згладжування сезонної компоненти. |
| `trend` | за замовч. (None -> авто, непарне) | Вікно LOESS тренду; залишити авто, збільшити для гладшого тренду. |
| PELT `model` | `"rbf"` | Ядро для виявлення зсувів рівня/дисперсії. |
| PELT `pen` | `10` (стартове) | Штраф; підбирати, щоб кількість точок зміни була розумною (одиниці, не десятки). |

Мінімальна довжина ряду: щонайменше 2 повні періоди (>= 14 днів); на практиці датасет з 2022-02-25 дає сотні точок.

## Критерій якості

1. **Перевірка адитивності/реконструкції:** `trend + seasonal + resid` має відтворювати вхід:
   `np.allclose((trend + seasonal + resid).values, series.values, atol=1e-6)` -> `True`.
2. **Сила компонент** (Hyndman strength measures), як числовий показник якості декомпозиції:
   ```python
   Ft = max(0, 1 - resid.var() / (trend + resid).var())      # сила тренду  (0..1)
   Fs = max(0, 1 - resid.var() / (seasonal + resid).var())   # сила сезонності (0..1)
   ```
   `Fs` помітно > 0 підтверджує наявність тижневого патерну; `Ft` > 0 — наявність тренду.
3. **Залишки** мають бути приблизно центровані навколо нуля (`abs(resid.mean())` малий
   відносно std ряду) — інакше переглянути `period`/вікна.
